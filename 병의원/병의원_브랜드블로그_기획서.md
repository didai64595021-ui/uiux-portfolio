# [Prompt for Claude Code] Medical Brand Blog Spec.md

**Project Goal:**
Develop a trustworthy, blog-style brand website for a medical clinic. The design layout should reference `https://blog.outta.ai/` but be optimized for patient conversion and trust-building.

**Core Concept:** "Soft Intelligence" (Professional knowledge delivered softly).
**Target Audience:** Patients searching for symptoms and treatment reviews.

## 1. Tech Stack & Environment

* **Framework:** Next.js 14 (App Router)
* **Styling:** Tailwind CSS
* **Icons:** Lucide React (Clean, rounded strokes)
* **Font:** 'Pretendard' (Essential for Korean typography)

## 2. Design System (Critical)

* **Shape:** **Heavily Rounded.**
  * Cards: `rounded-2xl` (approx 16px).
  * Buttons: `rounded-full`.
  * Search Bar: `rounded-full`.
  * *Reason:* To reduce patient anxiety and convey a gentle, caring image.

* **Color Palette:**
  * **Background:** Warm White (`#FAFAFA`) or very light beige. Avoid sterile pure white (`#FFFFFF`) for the main bg.
  * **Primary:** Soft Teal (`#20B2AA`) or Muted Blue (`#4A90E2`) - Trustworthy but not cold.
  * **Text:** Dark Gray (`#333333`) for readability, Light Gray (`#888888`) for meta info.

* **Shadows:** Soft, diffused shadows (`shadow-lg` with low opacity) instead of harsh borders.

## 3. Layout & Components

### A. Header (Sticky)

* **Left:** Hospital Logo (Text + Icon).
* **Center:** Navigation (Medical Columns, Case Studies, Hospital News, About Us).
* **Right:**
  * Search Icon (Expandable).
  * **CTA Button:** "간편 예약/상담" (Primary Color, Pill shape).

### B. Hero Section (Top Area)

* **Style:** Minimalist. No large, heavy hero images.
* **Content:**
  * Greeting/Slogan: "환자의 마음까지 치료합니다."
  * **Keyword Filters:** A row of rounded toggle buttons (e.g., #임플란트, #치아교정, #통증치료). allow users to filter the grid immediately.

### C. Main Content (The Grid)

* **Layout:** Responsive Masonry or Grid (3 columns on Desktop, 1 column on Mobile).
* **Card Component:**
  * **Image:** Top half. High-quality treatment image or doctor's photo. (Apply `rounded-t-2xl`).
  * **Tag:** Floating badge on top-left of image (e.g., [치료사례]).
  * **Title:** Bold, `text-lg`.
  * **Excerpt:** 2 lines max, `text-gray-500`.
  * **Meta:** Doctor Name • Date.

* **Interaction:** Hovering over a card lifts it slightly (`-translate-y-1`) and deepens the shadow.

### D. Detail Page (Post View)

* **Container:** Centered, max-width 768px (Optimization for reading).
* **Typography:** Large font size (18px body) for elderly readability. Line-height 1.8.
* **Sticky Widget (Right Side - Desktop only):**
  * Doctor's Profile (Small circle img).
  * Operating Hours.
  * Map/Location.
  * "KakaoTalk Consultation" Button.

### E. Floating Action Button (Mobile Only)

* **Position:** Bottom Right, fixed.
* **Icon:** Calendar or Phone icon.
* **Action:** Triggers a modal for quick booking or dialing.

## 4. Specific Requirements for Implementation

1. **Mobile First:** Ensure the grid collapses gracefully to a single column on mobile screens.
2. **Loading State:** Use 'Skeleton Loaders' (shimmer effect) that match the rounded card shape while data is fetching.
3. **SEO:** Semantic HTML (`<article>`, `<section>`, `<h1>`) is mandatory for search engine exposure.

## ⛔ 냉철한 피드백 및 리스크 분석 (Critical Critique)

병의원 웹사이트로서 이 기획이 실패할 수 있는 요인과 기술적 보완점입니다.

1. **신뢰도 하락의 위험 (The "Amateur" Risk)**
   * **비판:** `blog.outta.ai`와 같은 텍스트 위주의 썸네일 없는 배치는 병원 사이트에서 치명적입니다. 환자는 '전후 사진'이나 '깨끗한 병원 내부', '의사의 얼굴'을 보고 싶어 합니다.
   * **수정:** 카드 컴포넌트에서 **이미지 비중을 50% 이상**으로 강제 설정하십시오. 텍스트만 있는 카드는 사용자 이탈을 유발합니다.

2. **목적성 상실 (Conversion Funnel)**
   * **비판:** 블로그 형식을 그대로 따르면 환자는 정보만 얻고 이탈(Bounce)합니다. 병원의 목적은 '예약'입니다.
   * **수정:** 상세 페이지(Post View)의 **본문 중간과 하단에 반드시 "이 증상 상담받기" 버튼**을 삽입해야 합니다. 콘텐츠 소비가 곧장 행동으로 이어지게 강제하십시오.

3. **가독성 이슈 (Accessibility)**
   * **비판:** 레퍼런스 사이트의 폰트 사이즈는 젊은 개발자 타겟이라 다소 작을 수 있습니다. 병원 주요 고객층(중장년층)에게는 작습니다.
   * **수정:** 기본 본문 폰트 사이즈를 **최소 16px, 권장 18px**로 설정하고, 명도 대비(Contrast)를 확실하게 주어 흐릿한 회색 글씨를 피하십시오.
